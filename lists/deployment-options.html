breakdown

numtide / llm-agents.nix

This project provides daily updated Nix packages for a wide array of AI coding agents, including Hermes. It allows for direct tool execution via Nix flake commands, making it a versatile choice for users managing multiple agent environments.

TheAiSingularity / hermesclaw

Hermesclaw offers a high-security deployment by utilizing hardware-level enforcement to maintain strict network, filesystem, and process boundaries. It includes three security policy presets and supports over 40 built-in tools, including MCP integration.

0xrsydn / nix-hermes-agent

Designed for reproducibility, this NixOS module enables declarative management of agent configurations, documents, and systemd services. It ensures secure secrets handling through sops-nix integration and supports both Nix-managed and interactive CLI installations.

rookiemann / portable-hermes-agent

This Windows-based solution provides a no-install, no-Docker deployment that requires no administrative rights. It features a GUI, 100 integrated tools, local model management, and a workflow engine for scheduled or conditional automation.

xmbshwll / hermes-agent-docker

This project offers a straightforward Docker sandbox image that includes the official Hermes installation and mini-swe-agent. Users can maintain persistent state through mounted home directories and build specific Hermes tags using build arguments.

JackTheGit / hermes-autonomous-server

Tailored for production environments on Ubuntu and Debian, this headless deployment configures Hermes Gateway as a reboot-safe systemd service. It is optimized for autonomous execution of recurring AI cron jobs and integrates with the Nous Portal.

ellickjohnson / portainer-stack-hermes

This deployment utilizes a Docker Compose stack that integrates directly with Portainer as a Git Repository stack. It features a ttyd web terminal interface and automates image builds via GitHub Actions and GHCR.
